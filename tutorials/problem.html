Création d'un problème

Ce tutoriel a pour objectif la création d'un problème. Nous dévelepperons donc en plusieurs parties :

  1. Se rendre dans la partie adéquate de l'administration
  2. Rappel de la préconfiguration d'un langage
  3. Définir les informations relatives à un problème
  4. Rédiger l'énoncé du problème
  5. Définir le code du problème
  6. Conseil de problèmes similaires

1. Se rendre dans la partie adéquate de l'administration

La partie à venir détaille les trois champs principaux que vous

Définir les librairies à importer : la préconfiguration du langage peut mettre à disposition des librairies, cependant si les librairies proposées ne permettent pas de résoudre votre problème, c'est dans cette partie qu'il faudra rajouter vos librairies supplémentaires.

2 ème étape :
Il s'agit ici du champ utilisé pour définir le code qui sera affiché dans la l'éditeur de l'étudiant pour la résolution du problème. Ce champ peut être prérempli ou bien être laissé vide. Dans ce cas, vôtre énoné devra être assez clair pour que l'étudiant puisse comprendre ce qu'il devra faire

Vous avez aussi la possibilé grâce au bouton "Comment" de placer un commentaire de type "TODO" afin d'indiquer les parties du code qui seront à implémenter


Image 7

3 ème étape :
La dernière étape de la création de code consiste à définir les tests qui valideront l'implémentation du problème par l'étudiant. Dans le but d'assurer une validation plus sûre de l'implémentation du problème, il est conseillé de produire plus d'un test unitaire.

Afin de faire fonctionner correctement le service de compilation à distance, il est nécéssaire d'utiliser au moins une fois chacune des balises "True" et "False" (Flèches 2 et 3). Elles permettront de faire savoir au plugin si le code exécuté donne le bon résultat ou non.

Pour terminer, vous pouvez décider d'afficher dans la page de résolution ou non les portions de code contenant les tests unitaires. Si vous décidez d'afficher une portion de code, il suffira de sélectionner la zone à afficher, puis de cliquer sur le bouton "Unit Test" (Flèche 1). Par example, sur l'image 7, seulement les tests unitaires des lignes 2 et 4 seront visibles par l'étudiant.
